Java基本语法

Posted on 2016-03-19

Java基本语法

  • 区分大小写
  • 定义多个Java类,但其中最多只能有一个类被动定义成public类.如果原文件中包括了public类,类名和文件名必须相同.

变量分类-作用域-使用规则:

变量的分类

  • 变量根据定义的位置不同,分成两大类

    1:成员变量/字段:

  • 直接定义在类中(方法外).

    2:局部变量:

  • 变量除了成员变量就是局部变量.

变量的作用域

作用域:可以起作用的领域

  • 根据变量所在的{}来判断

    1:成员变量:

  • 在所定义的类中起作用.

    2:局部变量:

  • 从定义的那一个地方开始,到紧跟着结束的}之间.

字符串

  • 字符串的拼接:字符串和任意数据类型相连接,结果都是字符串类型.

构造器

构造器的作用

  • 创建对象,凡是必须和new一起使用.
  • 完成对象的初始化操作.

构造器的特点:

  • 构造器的名称和当前所在类的名称相同.

  • 禁止返回类型.千万不要使用void作为返回类型.

  • 假设需要写返回类型,也应该这样写

  • 既然所有的返回类型都和所在的类名相同,所以不必再写.

  • 在构造器中,不需要使用return语句.其实构造器返回值的,返回的当前创建对象的引用.

编译器创建的默认构造器的特点

  • 符合构造器的特点.
  • 无参数的.
  • 无方法体.
  • 如果类A没有使用public修饰,则编译器的构造器也没有public修饰.

自定义构造器

  • 如果我们没有显示提供构造器.则编译器在编译时创建一个缺省的构造器.但是如果我们显示定义了一个构造器,则编译器不在创建默认的构造器.
  • 某一个类,至少存在一个构造器

static修饰符

  • 生活中有些东西不是属于某一个具体的对象的,而是属于整个事物的类型,比如:全世界人额总数,人的毁灭的行为.

状态和行为应该有对象和类型之分.

  • 有的状态和行为属于对象,不同的对象,状态和行为可以不一样.
  • 有的状态和行为应该属于类型,不属于对象.

问题:现在如何表示人类毁灭这个行为(destory).
要解决该问题,得学习static修饰符.

修饰符

  • 晚上11点,班长在写代码,遇到一个bug,班长解决了问题后,睡觉去了.
  • 晚上11点,班长漫不经心地在写代码,遇到一个简单的bug,班长稀里糊涂的解决了问题后,满不在乎的睡觉去了.
  • 晚上11点,帅气的班长认认真真地在写代码,遇到一个超难的bug,班长呕心沥血解决问题之后,眉开眼笑的睡觉去了.
  • static修饰符表示静态的,可以修饰字段,方法,内部类,其修饰符的成员属于类,也就是说static修饰的资源属于类级别,而不是对象级别.
  • staitc的真正的作用:用来区别字段,方法,内部类,初始化代码块是属于对象还是属于类本身.

static修饰符特点

  • static修饰的成员(字段/方法),随着所在类的加载而加载.
  • 优先于对象的存在.
  • static修饰的成员被该类型的所以对象所共享.(表面上通过对象去访问static成员,其本质依然使用类名访问,和对象没有任何关系)
  • 因为static修饰的成员直接属于类,不属于对象,可以直接使用类名访问staitc成员.

成员变量和静态变量的区别

1.两个变量的生命周期不同.
  • 成员变量随着对象的创建而存在,随着对象的被回收而释放.
  • 静态变量随着类的加载而存在,随着类的消失而消失.
